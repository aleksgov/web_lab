Язык разметки XML

"XML — это язык, который превратил хаос данных в стройную симфонию структурированной информации."
Неизвестный мыслитель

XML (Extensible Markup Language, расширяемый язык разметки) стал своего рода "золотой серединой" между языками SGML и HTML. Изначально он был создан как средство обмена документами в интернете, но со временем превратился в универсальный инструмент для хранения и передачи данных в виде иерархических структур. XML является упрощённым подмножеством SGML (Standard Generalized Markup Language, стандартный обобщённый язык разметки) — метаязыка, позволяющего создавать собственные языки разметки для документов (см. рис. 1). Сам язык появился в 1986 году.

Рис. 1. Подмножество языка GML

XML‑теги идентифицируют данные и используются для хранения и организации данных, а не указывают, как их отображать как HTML-теги, которые используются для отображения данных. XML не собирается заменять HTML в ближайшем будущем, но он открывает новые возможности, применяя многие успешные функции HTML.

Есть три важных характеристики XML, которые делают его полезным в различных системах и решениях:

  • XML расширяем — XML позволяет вам создавать свои собственные описательные теги или язык, который подходит вашему приложению.
  • XML переносит данные, не представляет их — XML позволяет хранить данные независимо от того, как они будут представлены.
  • XML является общедоступным стандартом — XML был разработан организацией под названием World Wide Web Consortium (W3C) и доступен в качестве открытого стандарта.

Использование XML

Краткий список использования XML говорит само за себя:

  • XML может работать за сценой, чтобы упростить создание документов HTML для крупных веб‑сайтов.
  • XML может использоваться для обмена информацией между организациями и системами.
  • XML можно использовать для выгрузки и перезагрузки баз данных.
  • XML может использоваться для хранения и упорядочения данных, что может настраивать ваши потребности в обработке данных.
  • XML можно легко объединить с таблицами стилей для создания практически любого желаемого результата.
  • Фактически, любой тип данных может быть выражен в виде XML‑документа.

Иерархическая модель данных и её особенности

Иерархическая модель данных представляет данные в виде древовидной структуры. В такой базе данных сущности могут иметь родительские и дочерние связи, но обязательно существует одна корневая сущность, которая не имеет родителя, и все остальные элементы являются её потомками (см. рис. 2). Иерархические базы данных оптимизированы для быстрого чтения информации.

Что такое разметка?

XML — это язык разметки, который определяет набор правил для кодирования документов в формате, читаемом человеком и машиной. Так что же такое язык разметки? Разметка — это информация, добавляемая в документ, которая определенным образом усиливает его значение, поскольку она идентифицирует части и то, как они связаны друг с другом. Более конкретно, язык разметки — это набор символов, которые могут быть помещены в текст документа для разграничения и маркировки частей этого документа.

XML‑документ состоит из сущностей, которые могут содержать символьные данные и элементы разметки. К разметке относятся теги (строки текста, заключенные в угловые скобочки, например, < >), которые обозначают границы элементов, атрибуты этих элементов и комментарии. В отличие от HTML, XML не имеет фиксированного набора тегов, а предоставляет правила для создания собственных. Хорошо оформленный XML-документ — это документ, который строго следует этим правилам.

XML — синтаксис

В начале каждого XML‑документа располагается пролог, который задает основные параметры документа. Пролог обязательно включает объявление XML для версий языка старше 1.0, что выглядит следующим образом:

<?xml version="1.0" encoding="UTF-8"?>

Атрибут version="1.0" указывает на версию XML, а атрибут encoding="UTF-8" — на кодировку документа, которая гарантирует правильное отображение символов. В прологе также могут присутствовать дополнительные инструкции, такие как DOCTYPE, которая позволяет с помощью языка схем DTD описать структуру документа, включая элементы, их атрибуты и допустимые способы вложенности. Также в прологе могут быть инструкции обработки, например, для подключения таблицы стилей CSS:

<?xml-stylesheet href="style.css"?>

В XML‑документе могут встречаться комментарии, начинающиеся с <!‑‑ и заканчивающиеся на ‑‑>. Эти комментарии игнорируются при обработке документа, но служат для пояснений и организации структуры файла.

Существуют строгие правила для правильного построения XML‑документа:

  1. Документ должен начинаться с объявления XML.
  2. Он должен содержать только один корневой элемент.
  3. Все теги, открытые в документе, должны быть обязательно закрыты.
  4. Имена элементов и атрибутов не могут содержать пробелы, и они регистрозависимы.
  5. Перекрытие элементов в документе недопустимо, необходимо соблюдать правильную вложенность элементов.

XML-документ также может содержать специальные символы, которые необходимо экранировать, чтобы избежать их путаницы с синтаксисом XML. Для этого используются такие экранированные формы, как < для символа "<", > для ">", и так далее. (см. рис. 3)

Рис. 2. Экранирование специальных символов в XML

Весь XML-документ состоит из разметки (например, <contact‑info>) и текстовых данных (например, Tanmay Patil или (011) 123‑4567). Пример правильного XML‑документа: 

<?xml version="1.0"?>
<contact-info>
    <name>Tanmay Patil</name>
    <company>TutorialsPoint</company>
    <phone>(011) 123‑4567</phone>
</contact‑info>

Здесь <contact-info> является корневым элементом, а внутри него находятся элементы <name>, <company> и <phone>, каждый из которых содержит текстовую информацию.

Является ли XML языком программирования?

Язык программирования состоит из грамматических правил и собственного словарного запаса, который используется для создания компьютерных программ. Эти программы инструктируют компьютер для выполнения определенных задач. XML не может считаться языком программирования, поскольку он не выполняет никаких вычислений или алгоритмов. Обычно он хранится в простом текстовом файле и обрабатывается специальным программным обеспечением, способным интерпретировать XML.
