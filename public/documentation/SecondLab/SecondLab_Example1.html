Задание для первой части

Цель данного задания — создать корректный XML-документ, который будет описывать иерархическую структуру базы данных, содержащую несколько типов вложенных элементов и не менее четырех уровней вложенности. Этот документ должен включать различные атрибуты и текстовые значения, соответствующие концепции базы данных, которую вы изучали в рамках курса "Базы данных". В качестве примера, можно использовать схему базы данных, связанную с организацией соревнований по карате, как основу для создания XML-документа.

Отчет должен содержать:

  • Листинг XML-документа.
  • Описание элементов XML-документа.
  • Вывод о проделанной работе.

Описание задачи

Задача заключается в составлении XML-документа, который описывает структуру базы данных (БД) для соревнований по карате. Важно, чтобы структура XML-документа имела иерархию, включала различные типы элементов, такие как участники, судьи и секретариат, и соответствовала требованиям по вложенности. Элементы должны содержать атрибуты и текстовые значения, отражающие такие характеристики, как категория участников, результаты боев и другие данные.

Структура XML-документа

Для реализации данного задания можно использовать следующую структуру XML-документа:

  • Корневой элемент: <karateCompetitions> — содержит все данные, связанные с соревнованиями по карате.
  • Элементы татами: <tatami> — описывают каждый ковёр, на котором проводятся соревнования.
  • Судейская команда: <judgingTeam> — описывает команду судей, работающих на каждом татами.
  • Секретариат: <secretariat> — включает информацию о членах секретариата каждого татами.
  • Бои: <kumite> — представляет информацию о проведенных боях, включая результаты и участников.
  • Выступления (ката): <kataPerformances> — описывает выступления участников, включая их оценки.

Написание XML-документа

На основе описанной структуры, XML-документ будет содержать вложенные элементы. Каждый элемент будет иметь атрибуты, которые позволяют точно определить его характеристики:

1. Корневой элемент

Документ начинается с корневого элемента <karateCompetitions>, который включает в себя все данные, описывающие соревнования.

<karateCompetitions>
</karateCompetitions>

Все остальные элементы будут находиться внутри этого корневого.

2. Элементы татами

Каждое татами, используемое для проведения соревнований, представлено вложенным элементом <tatami>. Для идентификации татами используются уникальные атрибуты id (например, "A", "B").

<karateCompetitions>
    <tatami id="A">
    </tatami>
    <tatami id="B">
    </tatami>
</karateCompetitions>

3. Судейская команда

Каждое татами включает элемент <judgingTeam>, содержащий данные о судейской команде. Вложенные элементы <judge> описывают каждого судью, включая уникальный идентификатор, категорию судейства и текстовые значения имени.

<tatami id="A">
    <judgingTeam id="1">
        <judge id="1" category="1-ая категория">
            <name>
                <lastName>Данилов</lastName>
                <firstName>Станислав</firstName>
            </name>
        </judge>
    </judgingTeam>
</tatami>

4. Секретариат

Элемент <secretariat> содержит сведения о членах секретариата, включая их идентификаторы и текстовые значения имени.

<secretariat>
   <member id="1">
       <name>
           <lastName>Носов</lastName>
           <firstName>Егор</firstName>
       </name>
   </member>
</secretariat>

5. Бои (кумите)

Элемент <kumites> включает данные о боях. Каждый бой представлен элементом <kumite> с уникальным идентификатором, результатом и списком участников. Участники описываются вложенными элементами <participant> с информацией о категории, возрасте, весе и регионе.

<kumites>
   <kumite id="24">
       <decision>победа красного</decision>
       <participants>
           <participant id="327" side="белый">
               <name>
                   <lastName>Моргунов</lastName>
                   <firstName>Дмитрий</firstName>
               </name>
               <category>
                   <age>14-15 лет</age>
                   <weight>65-70 кг</weight>
                   <gender>м</gender>
               </category>
               <region>Нижегородская область</region>
           </participant>
       </participants>
   </kumite>
</kumites>

6. Выступления (ката)

Элемент <kataPerformances> описывает выступления участников. Каждый элемент <kata> содержит данные об участнике и его оценке.

<kataPerformances>
   <kata id="12">
       <participant id="327">
           <name>
               <lastName>Моргунов</lastName>
               <firstName>Дмитрий</firstName>
           </name>
           <score>7.5</score>
       </participant>
   </kata>
</kataPerformances>

Уровни вложенности

В документе предполагается следующая структура вложенности:

  • Первый уровень: <karateCompetitions>.
  • Второй уровень: <tatami>.
  • Третий уровень: <judgingTeam>, <secretariat>, <kumites>.
  • Четвертый уровень: <judge>, <participant>, <kata>.

Заключение

В результате выполнения задания будет создан XML-документ, описывающий данные о соревнованиях по карате с четко структурированными иерархическими элементами, включая судей, участников, результаты боёв и ката. Каждый элемент будет содержать необходимые атрибуты и текстовые значения, что позволяет представлять данные в формате XML для дальнейшего использования в базе данных или других приложениях.
