# DataLab — учебная платформа лабораторного практикума по дисциплине «Управление данными»

DataLab — это веб-приложение для прохождения лабораторного практикума по дисциплине «Управление данными». Приложение предоставляет единый интерфейс для работы с теоретическим материалом, разобранными примерами и индивидуальными заданиями по вариантам. Учебный контент хранится в виде статических HTML-документов и подаётся клиенту по запросу, что позволяет дополнять и изменять материалы без пересборки фронтенда.

## Архитектура

Приложение состоит из двух частей:

- Клиент — одностраничное приложение на React (Vite), отвечающее за навигацию, отображение контента и пользовательский интерфейс.
- Сервер — приложение на Express (`server.js`), отдающее учебный контент и собранный фронтенд.

Контент организован по двум уровням:

- `content-registry.json` — реестр, сопоставляющий идентификатор материала (например, `lab1_theory`) с путём к HTML-файлу в каталоге `public/documentation/`. Сервер отдаёт только файлы, перечисленные в реестре, и проверяет итоговый путь, исключая выход за пределы `public/` (защита от обхода каталога).
- `materials/labN/manifest.json` — манифест каждой лабораторной работы, описывающий её состав: заголовок, идентификатор теории, список примеров, задания и количество вариантов.

Схема разделяет логику приложения и учебные материалы: добавление новой работы или варианта сводится к размещению HTML-файлов и правке реестра и манифеста.

## Возможности

- Навигация по пяти лабораторным работам через систему вкладок.
- Для каждой работы доступны разделы: теория, разобранные примеры и задания по вариантам.
- Открытые вкладки и активная вкладка сохраняются в `localStorage` и восстанавливаются при следующем входе (ограничение — до 10 вкладок).
- Подсветка синтаксиса в примерах кода (highlight.js).
- Выбор цветовой темы оформления с сохранением выбора в `localStorage`.

## Технологический стек

- Клиент: React 19, Vite, CSS Modules, highlight.js.
- Сервер: Node.js, Express.
- Менеджер пакетов: pnpm.

## Структура проекта

```
├── server.js                 — Express-сервер: раздача контента и собранного клиента
├── content-registry.json     — реестр идентификаторов контента и путей к HTML-файлам
├── index.html                — точка входа Vite
├── vite.config.js            — конфигурация Vite (прокси /content и /materials на сервер)
├── public/
│   └── documentation/        — HTML-файлы теории, примеров и заданий по работам
├── materials/
│   └── labN/manifest.json    — манифесты лабораторных работ
└── src/
    ├── App.jsx               — корневой компонент, маршрутизация по вкладкам
    ├── components/           — компоненты интерфейса (вкладки, аккордеон, выбор темы и др.)
    ├── hooks/                — хуки загрузки манифестов, контента и управления вкладками
    └── config/               — конфигурация (цветовые темы)
```

## Установка и запуск

Требуется установленный Node.js и пакетный менеджер pnpm.

1. Установка зависимостей:
```bash
pnpm install
```

### Режим разработки

Запускает сервер Express и dev-сервер Vite одновременно:

```bash
pnpm dev
```

Сервер контента поднимается на порту `3001`, клиент — на dev-сервере Vite. Запросы к `/content` и `/materials` проксируются на сервер контента. Для работы следует открыть адрес, указанный в выводе Vite.

### Режим production

1. Сборка клиента:
```bash
pnpm build
```
2. Запуск сервера, отдающего собранное приложение:
```bash
pnpm serve
```
3. Открыть в браузере:
```
http://localhost:3001
```

## Совместимость

Приложение кроссплатформенное и работает в средах Windows, Linux и macOS при установленном Node.js.
